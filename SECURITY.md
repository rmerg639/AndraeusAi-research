# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 2.x.x   | Yes                |
| 1.x.x   | No                 |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly:

**Email:** andraeusbeats@gmail.com

**Subject:** [SECURITY] Andraeus AI Vulnerability Report

**Include:**
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Response Timeline

- **Acknowledgment:** Within 48 hours
- **Initial Assessment:** Within 7 days
- **Resolution:** Dependent on severity

## Security Best Practices

When using this software:

1. **API Keys:** Never commit API keys or credentials
2. **Model Weights:** Store trained adapters securely
3. **Personal Data:** Encrypt personal training data
4. **Access Control:** Limit access to trained models
5. **Audit Logs:** Maintain logs of model usage

## Prohibited Security Uses

This software may NOT be used for:
- Unauthorized access to systems
- Breaking encryption
- Surveillance without consent
- Any illegal hacking activities

Violations will result in immediate license termination and legal action.
